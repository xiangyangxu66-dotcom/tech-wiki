Dropzone 処理フロー & 「停止」問題の原因

python manage.py dropzone --once | 23 × 0.07s = 1.6s（実処理）vs ユーザー体感「永遠に止まってる」

問題: OutputWrapper バッファ

  • • Django OutputWrapper.write() は内部バッファを持つ
  • • 非TTY環境（パイプ経由）では flush() が効かない
  • • 8KB バッファが満杯になるまで出力されない
  • • 23件（~3KB出力）→ 満杯にならず詰まる
  • • ユーザー視点：「真っ暗で止まった」

実性能: 問題なし

  • • Django起動: 0.30s（固定）
  • • 1ファイル処理: ~0.07s
  • • うちDB write: ~0.025s（35%）
  • • 23ファイル: ~1.6s で完了
  • • ボトルネック: SQLite FTS trigger

修正: 2行変更

  • • --once オプション追加（即終了）
  • • write() を monkey-patch
  • • 毎回 self.stdout._out.flush()
  • • 結果: リアルタイム出力 + 即終了
  • • 変更行数: ~15行
